# دليل إدارة وتحديثات تطبيق دنانير (Expo Updates)

هذا الملف يشرح كيفية استخدام نظام التحديثات عبر الهواء (OTA) الذي تم تفعيله مؤخراً، وكيفية رفع نسخ جديدة للمستخدمين بدون الحاجة لرفع تطبيق جديد للمتاجر.

## 1. المتطلبات الأساسية
يجب أن يكون لديك `eas-cli` مثبتاً على جهازك. إذا لم يكن لديك، قم بتنصيبه:
```bash
npm install -g eas-cli
```
وتأكد من تسجيل الدخول لحساب Expo الخاص بك:
```bash
eas login
```

## 2. كيفية رفع تحديث جديد (OTA)
عندما تقوم بتعديل الكود (JS) أو الصور/الأصول وتريد إرسالها للمستخدمين فوراً:

### الطريقة التلقائية (الأكثر استخداماً):
هذا الأمر سيقوم بعمل Bundle ورفعه للقناة الحالية (عادة `production`):
```bash
eas update --auto
```

### الرفع لمنصة محددة (iOS أو Android فقط):
```bash
eas update --platform ios --auto
# أو
eas update --platform android --auto
```

### الرفع مع رسالة وصفية (للسجلات):
```bash
eas update --message "إصلاح مشكلة الإشعارات وإضافة ميزات جديدة"
```

---

## 3. كيف يعمل النظام الآن؟
1. **عند فتح التطبيق**: سيقوم التطبيق بالتحقق من وجود تحديث جديد في الخلفية.
2. **عند توفر تحديث**: سيظهر للمستخدم مودال (Modal) يخبره بوجود تحديث جديد.
3. **عند الضغط على "تحديث الآن"**: سيقوم التطبيق بتحميل التحديث الجديد وإعادة التشغيل (Reload) فوراً بالنسخة الجديدة.

---

## 4. ملاحظات هامة جداً
- **تغييرات Native**: إذا قمت بإضافة "Plug-in" جديد يتطلب تغييرات في ملفات الـ Android أو iOS (مثل إضافة مكتبة كاميرا أو بلوتوث جديدة)، فإن التحديث عبر الهواء (OTA) **لن يعمل**. في هذه الحالة يجب عليك رفع "Build" جديد للمتاجر (Google Play / App Store).
- **Runtime Version**: النسخة الحالية مثبتة على `1.0.0` في ملف `app.json`. لا تقم بتغييرها إلا إذا كنت ستقوم بعمل Build جديد للمتاجر، لأن تحديثات OTA تعمل فقط للنسخ التي تملك نفس الـ Runtime Version.

---

## 5. الأوامر المفيدة الأخرى
- **لرؤية تاريخ التحديثات المرفوعة**:
```bash
eas update:list
```
- **للتراجع عن تحديث (Rollback) إذا حدث خطأ**:
```bash
eas update:rollback
```
